Casi studio anonimi

Problemi reali, senza nomi o dati sensibili.

E-commerce

Tracking incoerente tra varianti

Problema
La variante generava eventi duplicati sulle CTA renderizzate dopo il caricamento iniziale.
Soluzione
Controlli DOM, flag anti-duplicazione e nomenclatura eventi più esplicita in Data Collection.
Risultato
Dataset più leggibile e confronto tra varianti meno rumoroso durante l'analisi.

Lead generation

Metrica primaria poco stabile

Problema
La metrica scelta era troppo distante dall'interazione modificata dall'esperimento.
Soluzione
Separazione tra metrica primaria, metriche secondarie e guardrail di qualità del lead.
Risultato
Decisione finale più chiara, con meno dipendenza da oscillazioni di conversione a valle.

Single page app

Experience applicata troppo presto

Problema
Adobe Target eseguiva la modifica prima che il componente principale fosse disponibile nel DOM.
Soluzione
MutationObserver limitato al contenitore rilevante, timeout e disconnessione dopo l'applicazione.
Risultato
Experience più stabile su navigazioni dinamiche e minore rischio di flicker o doppie modifiche.

Come ho costruito questo sito

Questo sito è volutamente semplice: HTML semantico, CSS moderno e JavaScript puro. Non usa framework, build step o dipendenze esterne. La struttura è pensata per restare leggibile nel tempo e per funzionare direttamente su GitHub Pages.

  • HTML per contenuto e struttura delle sezioni.
  • CSS responsive con variabili, griglie e supporto dark mode.
  • JavaScript per articoli dinamici, filtri, ricerca, progress bar e demo tracking.
  • GitHub Pages come hosting statico, senza pipeline di compilazione.
